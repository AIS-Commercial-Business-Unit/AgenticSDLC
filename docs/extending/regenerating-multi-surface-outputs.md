# Regenerating Multi-Surface Outputs

## Background

The AIS Agentic Engineering Framework generates agent definition files for multiple AI surfaces from a single set of canonical source prompts in `.specify/prompts/`.

Previously, the following surface-specific directories were maintained as generated outputs:
- `.claude/` — Claude Code command files
- `.cursor/` — Cursor rule files
- `.agents/` — Standalone agent definition files (non-GitHub-Copilot)

These directories have been **removed from the source repository**. They are now treated as derived outputs that teams generate locally for their preferred surface(s).

The `.github/agents/` directory remains in the source repository as the primary agent surface for GitHub Copilot.

---

## Why This Change Was Made

- Maintaining identical content across multiple surface directories created synchronization drift.
- Different teams use different AI surfaces — forcing all surfaces into the repo added noise for teams using only one.
- The canonical source is the prompt. Surface-specific formats are transformations.
- Reduces repository size and merge conflicts in shared frameworks.

---

## Supported Surfaces

| Surface | Output Directory | File Format | Generation Method |
|---|---|---|---|
| GitHub Copilot (primary) | `.github/agents/` | `.agent.md` | Hand-authored or generated (in source repo) |
| Claude Code | `.claude/commands/` | `.md` + YAML frontmatter | Generated by `scripts/generate-commands.sh` |
| Cursor | `.cursor/rules/` | `.mdc` | Generated by `scripts/generate-commands.sh` |
| Standalone Agents | `.agents/` | `.md` | Generated by `scripts/generate-commands.sh` |

---

## How to Regenerate

Use the generation script to produce surface-specific outputs from source prompts.

### Prerequisites

```bash
# Ensure you have bash available
bash --version
```

### Generate All Surfaces

```bash
bash scripts/generate-commands.sh --all
```

### Generate a Specific Surface

```bash
# Claude Code commands
bash scripts/generate-commands.sh --surface claude

# Cursor rules
bash scripts/generate-commands.sh --surface cursor

# Standalone agent files
bash scripts/generate-commands.sh --surface agents
```

### Generate for a Specific Prompt

```bash
bash scripts/generate-commands.sh --surface claude --prompt brownfield.assess
```

## How the Generation Works

The generation script:

1. Reads each source prompt from `.specify/prompts/*.md`.
2. Reads the corresponding `.claude.yaml` frontmatter file (if present).
3. Applies a surface-specific template transformation.
4. Writes the generated output to the appropriate directory.
5. Reports any prompts that lack a surface-specific template.

### Transformation Rules

**Claude Code (`.claude/commands/`)**
- Takes the `.claude.yaml` frontmatter (`description`, `allowed_tools`, `output_type`).
- Copies the prompt body with the frontmatter prepended.
- Output file name: `[prompt-name].md` (removes the `brownfield.` prefix for Claude commands).

**Cursor (`.cursor/rules/`)**
- Extracts the first `## Purpose` and `## Instructions` sections from the prompt.
- Wraps in Cursor `.mdc` format.
- Output file name: `[prompt-name].mdc`.

**Standalone Agents (`.agents/`)**
- Copies the full prompt as-is.
- Adds a `<!-- Generated from: .specify/prompts/[name].md -->` header.

## Adding a New Target Surface

To add support for a new AI surface:

1. **Create a surface template** in `scripts/templates/[surface-name]/`.
   - The template defines how to transform a source prompt for this surface.
   - Template variables: `{{description}}`, `{{allowed_tools}}`, `{{prompt_body}}`, `{{prompt_name}}`.

2. **Add a case** in `scripts/generate-commands.sh` for the new surface.

3. **Document the surface** in this file under "Supported Surfaces."

4. **Update `.gitignore`** if the generated output directory should not be committed:
   ```gitignore
   # Generated surface outputs (regenerate with scripts/generate-commands.sh)
   .claude/
   .cursor/
   .agents/
   ```

---

## What Goes in .gitignore

Add these patterns to `.gitignore` if your team generates surfaces locally rather than committing them:

```gitignore
# Generated AI surface outputs — regenerate with scripts/generate-commands.sh
# Remove these lines if your team commits generated outputs
.claude/
.cursor/
.agents/
```

The `.github/agents/` directory is **always committed** — it is the primary surface and is not generated.

---

## Keeping Generated Outputs in Sync

When you update a source prompt in `.specify/prompts/`, run the generation script to keep all surface outputs current:

```bash
# After editing a prompt
bash scripts/generate-commands.sh --all
git add .claude/ .cursor/ .agents/
git commit -m "Regenerate surface outputs from updated source prompts"
```

Consider adding the generation script to a pre-commit hook or CI workflow to enforce synchronization.
