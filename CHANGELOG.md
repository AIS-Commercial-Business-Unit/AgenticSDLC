## [0.18.0] - 2026-05-26

### Added

- Add rich proposal DOCX constructs ([#148](https://github.com/ais-internal/ais-spec/pull/148))

Adds rich semantic content blocks to the proposal document generator, expanding the format beyond plain paragraphs and bullets into structured, AIS-branded constructs.

**New content block types:**
- **Tables** with AIS-styled formatting (ais_1, ais_2, ais_3 styles), column definitions, and row roles (header, subheader, body, centered)
- **Callouts** for highlighted notes and key differentiators with title, body, and optional bullet lists
- **Figures** with image embedding (file path or base64), alt text, captions, and placeholder support for images TBD
- **Q&A blocks** for solicitation question/response formatting
- **Requirement blocks** for RFP/RFI requirement callouts (shall, should, may, misc)
- **Theme blocks** for win theme emphasis using the AIS theme style
- **Heading2 and Heading3** paragraph types for subsection hierarchy within proposal sections

**Template and schema updates:**
- Onboards the 2025.12 AIS proposal template source (.dotx) alongside the existing templatized cover page
- Extends the JSON schema with a oneOf discriminated union for all content block types
- Adds a rich example input (rich-template-input.json) for QA and skill documentation
- Updates SKILL.md, STYLES.md, and TEMPLATE-STRUCTURE.md references

**Bug fixes:**
- Populates TOC placeholder entries ({{section_N_heading}}) from input section headings; removes any unfilled placeholders
- Strips leading numbers from Heading 1 text to prevent doubling with the template's auto-numbering list style
- Skips horizontal rule artifacts (---) that leak from markdown into body content
