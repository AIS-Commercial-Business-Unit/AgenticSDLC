Why it exists

The Gap Between AI Capability and Enterprise Readiness

Most organizations deploy GitHub Copilot without governance infrastructure. The result is inconsistent outcomes, compliance exposure, and stalled adoption.

The Problem

  • No audit trail for AI-generated code or agent actions
  • Undefined autonomy boundaries — agents act without oversight
  • No structured onboarding path for existing brownfield repositories
  • Ad-hoc agent sprawl with no catalog or version control
  • Compliance gaps with no evidence of review or approval gates
  • Inconsistent SDLC execution across teams using AI tools

The Framework

  • Structured audit trail with every agent action recorded and attributable
  • L0–L3 autonomy model defines exactly how much authority each agent has
  • Brownfield initializer assesses and governs existing repos systematically
  • Agent catalog with roles, boundaries, versioning, and permission reviews
  • Approval gates and human checkpoints baked into every lifecycle phase
  • AIS Specify 9-step lifecycle enforced consistently across all teams

Quick Start

Up and running in three steps

The framework is configuration-driven and requires no build tools. Works with any existing GitHub repository.

1

Clone the repository

Get the framework onto your machine. The .framework/ directory contains all governance scaffolding.

terminal bash 
$ git clone https://github.com/your-org/agentic-engineering-framework.git
$ cd agentic-engineering-framework
# Review the framework structure
$ ls .framework/
2

Copy the example configuration

Start from the example governance config and customize for your organization's policies and autonomy requirements.

terminal bash 
$ cp .framework/examples/governance.yml governance.yml
$ cp .framework/examples/agent-catalog.yml agent-catalog.yml
# Edit governance.yml to set your org defaults
$ code governance.yml
3

Run the brownfield assessment

Point the assessment playbook at any existing repository to generate a scored gap report and prioritized adoption roadmap.

terminal bash 
# Run via GitHub Copilot (with the framework loaded as a space)
> @copilot Run the Repository Assessment playbook for /path/to/my-repo

# Or run the assessment script directly
$ bash .framework/playbooks/repository-assessment.sh --target ../my-repo
$ cat assessment-report.md
Enterprise SDLC

Built for Enterprise Scale

The framework enforces engineering discipline at every level — from individual branches to organization-wide policies. GitHub rulesets, PR standards, and structured review gates are first-class.

  • Protected branch rules and GitHub Ruleset templates included out of the box
  • Structured PR templates with required fields for AI-assisted changes
  • Code review gate requirements with senior engineer sign-off for L2+ agent output
  • GitHub Rulesets integration for organization-wide enforcement
  • Branch naming conventions tied to lifecycle phases and issue tracking
  • Conventional commit standards enforced via commit hooks and CI validation
