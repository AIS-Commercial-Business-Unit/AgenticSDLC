Why it exists

The Gap Between AI Capability and Enterprise Readiness

Most organizations deploy GitHub Copilot without governance infrastructure. The result is inconsistent outcomes, compliance exposure, and stalled adoption.

The Problem

  • No audit trail for AI-generated code or agent actions
  • Undefined autonomy boundaries — agents act without oversight
  • No structured onboarding path for existing brownfield repositories
  • Ad-hoc agent sprawl with no catalog or version control
  • Compliance gaps with no evidence of review or approval gates
  • Inconsistent SDLC execution across teams using AI tools

The Framework

  • Structured audit trail with every agent action recorded and attributable
  • L0–L3 autonomy model defines exactly how much authority each agent has
  • Brownfield initializer assesses and governs existing repos systematically
  • Agent catalog with roles, boundaries, versioning, and permission reviews
  • Approval gates and human checkpoints baked into every lifecycle phase
  • AIS Specify 9-step lifecycle enforced consistently across all teams
What It Provides

Everything you need to govern AI engineering at scale

Eight capabilities that give your organization a repeatable, auditable AI engineering practice from day one.

⚙️

AIS Specify Lifecycle

A structured 9-step SDLC from Intake through Learn. Every development activity maps to a phase with defined inputs, outputs, and agent roles. Consistent across every team and project.

🔐

Autonomy Levels (L0–L3)

A four-tier model that precisely defines how much authority an agent can exercise. From suggest-only (L0) to bounded autonomous (L3), with explicit human approval requirements at each tier.

📋

Governance Registry

A version-controlled registry of all governance decisions, agent configurations, and policy rules. Every policy change is tracked, reviewed, and auditable — not stored in a spreadsheet.

🗂️

Brownfield Repository Assessment

Evaluate any existing repository against the AIS Specify standard. Detect gaps, generate a maturity baseline, and produce a prioritized adoption roadmap — without starting from scratch.

🤖

Agent and Skill Catalog

A structured inventory of every agent and reusable skill in your organization. Each entry documents role, boundaries, assigned autonomy level, permissions, and version history.

📊

Metrics and Evidence

Evidence-based maturity scoring per dimension, with traceable sources. Agent adoption, governance health, and framework maturity reported from real repository signals — not guesses.

🧪

Experiment Framework

Define bounded AI experiments with explicit hypotheses, success criteria, risk controls, and rollback conditions. Required before granting L2+ autonomy to any agent or workflow.

📦

Enterprise Starter Kit

Governance scaffolding, branch protections, PR templates, and CI policies ready to install. Teams go from zero governance to a structured, auditable workflow in a single command.

How It Works

Four stages from requirement to measured outcome

Every feature moves through four business-facing stages, each underpinned by the detailed AIS Specify lifecycle and governed agent activities.

01

Discover & Specify

Understand the repository, gather context, clarify business needs, and establish a measurable baseline. Structured specifications are committed to source control before any implementation begins.

AIS Specify steps: Intake · Specify

02

Architect & Plan

Architecture decisions, dependency analysis, risk assessment, and task decomposition. Human approval is required before any implementation begins, ensuring accountability at the highest-impact point.

AIS Specify steps: Design · Plan

03

Build & Verify

Bounded agent-assisted development, automated test generation, security validation, structured code review, and evidence collection. Agent autonomy is gated by the registered autonomy level.

AIS Specify steps: Implement · Verify

04

Deploy, Measure & Learn

Controlled release, metrics collection, retrospective analysis, experiment decisions, and governance refinement. Outcomes feed back into the registry and improve future cycles.

AIS Specify steps: Deploy · Report · Learn

Enterprise Scaling

Centrally Maintained, Distributed by Design

The framework is maintained centrally by the AIS Commercial Business Unit and installed into individual repositories via a single command. Teams get shared agents, skills, playbooks, and governance defaults without duplicating infrastructure.

  • Install the framework into any repository in minutes
  • Shared agent catalog, skill library, and governance policies
  • Controlled upgrades with repository-level configuration preserved
  • Teams contribute improvements through pull requests to the central framework
  • Governance health and maturity aggregated across repositories
  • Enterprise reporting across teams, business units, and the organization

Inner-Source Model

→ Central framework repository owns agents, skills, governance defaults
→ npm run install-framework bootstraps any target repo
→ Repositories override configuration; upgrades preserve customizations
→ Teams contribute via PR; Scribe agent merges and publishes updates
→ Metrics collected per-repo and rolled up to org-level dashboard
